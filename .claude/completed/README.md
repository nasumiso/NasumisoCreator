# 完了した要件のアーカイブ

このディレクトリには、完了した要件（REQ-XXX）に関する全情報が保管されています。

## ディレクトリ構造

```
completed/
└── REQ-XXX/
    ├── requirement.md  # 要件の詳細
    ├── work-plan.md    # 作業計画とチェックリスト
    └── notes.md        # 作業固有の重要なメモ
```

## 各ファイルの内容

### requirement.md
- requirements.md から抽出された要件の詳細
- ステータス、作成日、完了日などのメタ情報
- 要求内容、必要な要素、ゴールなど

### work-plan.md
- 作業計画のチェックリスト（完了状態）
- テスト結果の記録
- 実装時の進捗状況

### notes.md
- その要件に関連する重要な技術メモ
- 実装時に発見した問題点と解決方法
- 将来の参考になる情報

## 移動処理の流れ

要件が完了したら（Step 6完了後）：

### 1. 開発者が指示
```
「REQ-XXXを完了フォルダに移動してください」
```

### 2. Claude Codeが実行
1. `completed/REQ-XXX/` フォルダを作成
2. requirements.md から該当要件を抽出して `requirement.md` として保存
3. work-plan.md の内容を `work-plan.md` として保存
4. notes.md から関連する重要な情報を抽出して `notes.md` として保存
5. requirements.md と work-plan.md から該当部分を削除

### 3. 結果
- requirements.md: 進行中・未着手の要件のみ
- work-plan.md: 空（または次の作業待ち）
- completed/REQ-XXX/: 完了した要件の全情報

## アーカイブの活用

### 過去の実装を参照する
```bash
# 特定の要件を確認
cat completed/REQ-001/requirement.md

# 作業計画を確認
cat completed/REQ-001/work-plan.md

# 技術メモを確認
cat completed/REQ-001/notes.md
```

### 類似の作業を行う際
過去に完了した類似の要件を参照することで、効率的に作業を進めることができます。

### プロジェクトの振り返り
completed/ ディレクトリ全体を見ることで、プロジェクトの進捗と成長を振り返ることができます。

## 保持期間

- 基本的に無期限で保持
- プロジェクト完了時にアーカイブ化を検討
- 必要に応じて古い要件を別途バックアップして削除可能

## 命名規則

### REQ-XXX の番号付け
- REQ-001, REQ-002, ... のように連番で管理
- 0埋めは3桁を推奨（REQ-001 〜 REQ-999）
- 特殊な要件の場合は、接頭辞を追加可能（例: HOTFIX-001, REFACTOR-001）

## 注意事項

- このディレクトリのファイルは基本的に読み取り専用として扱う
- 修正が必要な場合は、新しい要件として作成することを推奨
- Git で管理することで、変更履歴も追跡可能
