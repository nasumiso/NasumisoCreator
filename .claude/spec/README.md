# 仕様書ディレクトリ

このディレクトリには、プロジェクトの仕様書を格納します。

## 基本ファイル

### implementation.md（必須）
実装済み機能を記録するファイルです。Claude Code が実装を完了するたびに、このファイルに実装内容を追記します。

### overview.md（推奨）
プロジェクトの概要、目的、機能概要などを記述します。プロジェクトの全体像を把握するための重要なファイルです。

## プロジェクトに応じた追加ファイル

プロジェクトの性質に応じて、以下のような仕様書を追加できます：

### Webアプリケーション
- `architecture.md` - システムアーキテクチャ
- `api.md` - API仕様
- `database.md` - データベース設計
- `ui-ux.md` - UI/UX設計

### ライブラリ/パッケージ
- `api-reference.md` - API リファレンス
- `design-decisions.md` - 設計判断の記録

### ゲーム開発
- `story.md` - ストーリー構成
- `characters.md` - キャラクター設定
- `systems.md` - ゲームシステム仕様

### モバイルアプリ
- `screens.md` - 画面構成
- `navigation.md` - ナビゲーション設計
- `data-flow.md` - データフロー

## 仕様書の管理方針

### 更新タイミング
- **初期作成**: プロジェクト開始時に開発者またはClaude Codeが作成
- **実装後**: Claude Codeが `implementation.md` に実装内容を記録
- **随時**: 開発者が必要に応じて仕様を修正・更新

### 保持ルール
- 常に最新の仕様を保持
- マイルストーン時のスナップショットは別途保存可能

### ファイル命名規則
- 小文字とハイフンを使用（例: `api-reference.md`, `data-flow.md`）
- 内容が明確にわかる名前にする

## 使い方

### 1. プロジェクト開始時
必要な仕様書ファイルを作成します。最小構成は以下の2ファイル：
- `overview.md` - プロジェクト概要
- `implementation.md` - 実装記録

### 2. 実装中
Claude Code が実装を完了するたびに `implementation.md` に記録します。

### 3. 仕様変更時
開発者が該当する仕様書ファイルを更新します。

## テンプレートファイル

このディレクトリには以下のテンプレートファイルが含まれています：
- `overview.md` - プロジェクト概要のテンプレート
- `implementation.md` - 実装記録のテンプレート

プロジェクトに応じて、これらをカスタマイズして使用してください。
